**`#ui <control>`** — spend EXTRA thought on a control before building/placing it; don't just drop a button where it's convenient. Answer first: (a) **purpose** — what one job does it do? (b) **frequency** — every session / sometimes / rarely / once? (c) **fit** — how does it sit beside the buttons already there (#cram: match the tightest existing density, one scrolling row not a wall; #design: small rounding, shared tokens)? Then place it by **REACH (the `#joy-of-less` ladder — more-used = closer to hand, rarer = further away):**

- **1 on-hand** = always visible, primary action on the main screen
- **2 pocket** = one tap (a visible toolbar pill / the graph bar)
- **3 backpack** = behind a fold/menu you open often
- **4 on table** = main Settings panel
- **5 shelf** = a deeper Settings sub-section
- **6 attic** = advanced / rarely-touched corner
- **7 warehouse** = not in the UI at all (config/code-only, or cut).

State the level you chose and WHY (its frequency), and prefer demoting a rare control DOWN the ladder over crowding level 1–2. A `#ui` is also a `#design` (sweep for siblings that belong at a different level). **Read `docs/ui-taste.md` first** — the owner's full UI taste profile (cram density + finger-friendly tap size + flat sharp pills + obvious affordance; what reliably annoys them; the reach/CEO tier lenses), distilled from the whole history.
